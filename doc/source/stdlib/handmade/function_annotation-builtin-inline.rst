Splices the function body into every direct call site during compilation, in every execution tier. Operator overloads (+, ==, +=, unary -, ...) qualify - their operator sites splice exactly like calls. This is a fail-closed contract, not a hint: a function shape the inliner can't splice, taking the function's address with @@, recursion through the inline graph, or a call in a global or field initializer is a compile-time error. Disable with options disable_inline (or CodeOfPolicies::disable_inline); the declaration-level contract checks (body shape, recursion, taking the address) still run, while call-site splice checks do not apply, since nothing splices.
